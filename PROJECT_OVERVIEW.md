# Inner Mirror — 基于内在家庭系统理论的多智能体 AI 心理疗愈系统

> **「每个人的内心都住着不同的声音，Inner Mirror 帮你看见它们、理解它们、整合它们。」**

---

## 一、产品简介

**Inner Mirror** 是一款基于 **内在家庭系统（Internal Family Systems, IFS）** 心理治疗理论打造的 AI 心理疗愈应用。IFS 由心理学家 Richard Schwartz 提出，认为人的内心由多个"子人格"（Parts）组成——管理者（Manager）、流放者（Exiles）、消防员（Firefighters），它们共同构成一个"内在家庭"。

本系统将这一前沿心理治疗理论与 **多智能体（Multi-Agent）AI 架构** 深度融合，构建了四个具有独立人格特征的 AI Agent：

| 角色 | 代号 | 职责 |
|------|------|------|
| 🔵 **管理者（Manager / Self）** | Manager | 用户的 Inner Voice，温暖地引导用户探索内心，负责整体调度 |
| 🟠 **流放者（Exiles）** | Exiles | 承载受伤、脆弱与被压抑需求的部分，渴望被看见和接纳 |
| 🔴 **消防员（Firefighters）** | Firefighters | 在情绪失控时迅速"灭火"与保护系统，代表理想化和使命感 |
| 🟣 **咨询师（Counselor）** | Counselor | 心理分析师，提取核心信念，生成深层心理洞察 |

用户只需与系统自然对话，系统即可通过 **实时情绪分析** 自动驱动人格之间的切换，在多轮对话中帮助用户探索内心冲突、识别深层核心信念，并通过 **内心议会辩论** 机制整合矛盾的内在声音，最终达到心理自我觉察与成长。

## 二、核心功能

### 2.1 🎭 多人格对话系统

- **Manager 主导对话**：用户始终与温暖的 Inner Voice 对话，系统在后台自动协调多个 Agent
- **实时人格切换**：情绪状态机根据对话中的情绪强度，自动在 Manager → Exiles / Firefighters 之间切换
- **沉浸式人格转场**：切换时展示全屏 Ripple 动画与人格信息，带来强烈的沉浸感
- **人格日记自动生成**：每次人格切换，对应子人格以第一人称视角书写意识流日记，配合 AI 生成意象图像

### 2.2 ⚖️ 内心议会辩论

- **结构化多轮辩论**：Exiles 与 Firefighters 围绕用户的核心信念冲突展开 2-3 轮辩论
- **Counselor 整合总结**：辩论结束后，咨询师 Agent 生成整合性心理洞察
- **实时推送**：辩论进展通过 SSE 实时推送到前端，用户可实时观看内心冲突的"辩论赛"
- **核心信念提取**：从议会结论中自动识别深层核心信念（如 "我不值得被爱"、"世界是不安全的"）

### 2.3 🧠 核心信念星图（Belief Map）

- **可视化信念系统**：以星座图形式展示用户被发现的核心信念
- **动态更新**：每次对话和议会后，新发现的信念以动画效果加入星图
- **强度与极性标注**：每条信念附带情感强度（0-10）和极性（正面/负面）评分

### 2.4 🖼️ 疗愈相册（Healing Album）

- **AI 绘图**：基于日记内容自动生成意象画面（调用 DashScope Wanx 模型）
- **用户反馈闭环**：用户可对日记写下感受，反馈会触发新一轮分析，推动疗愈进程
- **人格色彩标识**：每张卡片标注对应的人格类型与时间线

### 2.5 📖 沉浸式叙事（Immersive Mode）

- **时间线模式**：以逐帧叙事卡片形式回顾对话历程中的关键节点
- **故事模式**：AI 将用户的对话历程编织成连贯的章节式故事，配有章节标题与插图

### 2.6 🤝 灵魂共振社交系统

- **次人格相似度匹配**：基于用户核心信念向量的余弦相似度，匹配拥有相似内在冲突的"灵魂伙伴"
- **5 个预设 AI Agent 灵魂伙伴**：小星（温暖陪伴）、小海（深度探索）、小山（坚韧力量）、小风（自由灵魂）、小光（希望之光），各具独立人设与核心信念
- **漂流瓶系统**：用户可将次人格日记以漂流瓶形式投放，其他用户有机会捡到并共鸣
- **匿名 1v1 对话**：与 AI Agent 或真实用户进行匿名心理共鸣对话

### 2.7 📊 探索报告（Report）

- **Self-Presence 量化指标**：自我觉知清晰度（Clarity）与自我接纳（Compassion）双维度量化
- **趋势追踪**：记录每次对话的情绪变化曲线与人格切换历史
- **核心信念汇总**：全局视角展示用户所有已发现的核心信念及其演变

### 2.8 🔐 Git 式版本存档

- **SHA-256 哈希标识**：每个心理状态快照生成唯一哈希，支持精确回溯
- **版本树浏览**：以树状结构展示心理状态的演进分支
- **语义检索**：基于 ChromaDB 对历史快照进行语义相似度搜索

### 2.9 🛡️ 自动修复系统（AutoHeal）

- **21 种错误模式识别**：从容器日志中实时检测 Python 常见错误（ImportError、SyntaxError 等）
- **三级修复链路**：内置策略 → Gemini CLI AI 修复 → Claude Code CLI 终极兜底
- **安全机制**：60 秒冷却期、每小时最多 10 次、仅修改限定目录、修复前自动备份

## 三、技术方案特色

### 3.1 🧩 四 Agent 协同的多智能体架构

系统核心创新在于将 IFS 理论中的四个心理角色映射为四个独立的 AI Agent，每个 Agent 拥有精心设计的 System Prompt 和行为策略：

```
用户输入 → Manager（对话） → Counselor（分析） → Manager（调度决策）
                                                      ↓
                                           Exiles / Firefighters（激活）
                                                      ↓
                                              内心议会（辩论）
                                                      ↓
                                             Counselor（整合总结）
```

**协作流程**：
1. **Manager** 始终作为用户的唯一对话入口，以 Inner Voice 身份温暖倾听
2. **Counselor** 在后台对每次对话进行深度心理分析，提取核心信念和诱发事件
3. **Manager** 根据分析报告做出调度决策，决定是否激活子人格
4. 子人格激活后自动生成第一人称意识流日记 + AI 意象图像
5. **内心议会**中 Exiles 与 Firefighters 围绕核心信念展开多轮辩论
6. **Counselor** 最终给出整合性心理洞察

### 3.2 ⚙️ 情绪驱动的人格状态机（Emotion-FSM）

系统实现了一个精密的有限状态机来管理人格切换：

- **情绪强度阈值切换**：$I > 0.7$ 触发子人格激活，$I \leq 0.3$ 回归 Manager
- **波动性区分**：高波动 → Exiles（原始情绪爆发），低波动 → Firefighters（持续保护压力）
- **时间衰减机制**：子人格活跃时，情绪强度按公式 $I_{eff} = I_{raw} \times \max(0, 1 - t \times \gamma)$ 自然衰减
- **双模式情绪分析**：优先使用 DashScope API，降级时使用基于中文情感词典的规则引擎

### 3.3 🧠 RAG 长期记忆管线

基于 ChromaDB + Sentence Transformers 构建的向量记忆系统：

- **Embedding 模型**：all-MiniLM-L6-v2，支持 ModelScope / HF-Mirror / 本地缓存多级加载
- **混合检索**：语义相似度搜索 + 用户隔离过滤
- **记忆类型**：对话记忆、咨询师报告、日记文本均入向量库
- **上下文增强**：检索相关历史记忆注入 LLM Prompt，实现跨会话的连续心理分析
- **用户画像持久化**：自动提取并累积用户个人信息（性别、年龄、职业等），构建长期画像

### 3.4 🎨 WebGL Shader 情绪化背景

自研 GLSL Fragment Shader 作为全局动态背景：

- **人格色彩映射**：Manager（粉紫）、Exiles（暖橙）、Firefighters（冰蓝）、Counselor（淡紫）
- **fBm 噪声算法**：分形布朗运动生成有机流动纹理
- **粒子系统**：情绪强度驱动粒子数量和运动速度
- **呼吸动效**：低频正弦波模拟自然呼吸节律
- **移动端降级**：自动检测设备能力，移动端降低 fBm 迭代次数和粒子数量

### 3.5 📡 SSE 实时事件推送

基于 Server-Sent Events 构建的实时通信系统：

| 事件类型 | 说明 |
|----------|------|
| `persona_switch` | 人格切换通知 |
| `emotion_update` | 情绪强度更新 |
| `council_round` | 议会辩论进展 |
| `council_complete` | 议会辩论完成 |
| `counselor_report` | 心理分析报告 |
| `healing_image` | 疗愈日记 + 图像 |
| `belief_update` | 核心信念更新 |

前端通过 `useSSE` Composable 统一消费所有事件，自动处理断线重连、Token 刷新和健康检查。

### 3.6 🏗️ Glassmorphism 暗色设计系统

统一的沉浸式 UI 设计语言：

- **毛玻璃卡片**：`background: rgba(255,255,255,0.06); backdrop-filter: blur(12px)`
- **设计 Token 化**：颜色、间距、字体通过 Design System 模块统一管理
- **全屏沉浸式布局**：`position: fixed; inset: 0` 的根容器 + 流式滚动内容
- **人格化配色**：每个人格拥有独立的色彩方案，贯穿 UI 各处

### 3.7 🎓 Onboarding 个性化建模

首次登录时的两道简答题驱动个性化建模：

- **题1**：回忆一个印象深刻的情绪时刻 + 最不想让别人看到的情绪
- **题2**：压力大时内心的声音 + 对自己的描述

系统调用 LLM 从回答中提取：
- 个性化的 Exiles / Firefighters System Prompt
- 创伤线索假设
- 用户核心画像与核心信念
- 两个次人格的固定形象（不超过 90 字的人设描述）

后续所有对话均基于此画像进行个性化回应。

| 组件 | 要求 |
|------|------|
| Python | 3.11+ |
| Node.js | 18+ |
| Docker | Docker Compose V2 |
| LLM API | DashScope (Qwen-Max / Qwen-Plus) |
| 图像 API | DashScope (Wanx) |
| 嵌入模型 | all-MiniLM-L6-v2 (自动下载) |

### 快速启动

```bash
# 克隆项目
git clone https://www.modelscope.cn/studios/zhupeiling/MultiMe_IFS.git
cd MultiMe_IFS

# 配置环境变量
cp .env.example .env
# 编辑 .env，填入 DashScope API Key

# Docker 一键启动
docker compose up -d --build

# 访问
# 前端: http://localhost:5173
# 后端 API: http://localhost:8000/docs
```
